# 崩岸分析系统（前端）组会展示 PPT（Markdown 版）

> 说明：按“约 8 页 PPT”组织内容；每页控制在少量要点，覆盖当前前端全部交互功能，并包含架构与未来改进方向。

---

## 1/8 项目目标与前端定位

- 面向业务用户的“崩岸风险评估与可视化”前端：完成岸线/断面数据准备、参数配置、任务发起、结果可视化与追踪
- 核心交互围绕三步：**数据加载 → 断面设计/校验 → 任务分析与结果查看**
- 设计原则：地图交互优先（Mapbox），业务动作下沉（actions/api 模块化），前端尽量保持可解释与可追踪（任务/进度/错误明细）

---

## 2/8 技术栈（当前实现）

- 工程与语言：Vite + React + TypeScript（开发 HMR，生产构建 `tsc -b` + `vite build`）
- 地理/可视化：Mapbox GL JS（地图渲染与图层交互）、Turf.js（bbox、线段计算、相交/投影等几何工具）
- UI 与样式：CSS Modules（侧边栏/弹窗）+ 全局 CSS（导航/结果页样式）、Lucide React 图标
- 前后端联调：浏览器 `fetch`；Vite devServer 代理 `/v0 -> 后端服务`（避免跨域、统一接口前缀）

---

## 3/8 页面结构与用户主流程

- 首页（Home）：系统入口与引导，“开始使用”进入断面编辑器
- 断面编辑器（Editor）：岸段加载/选择 → 断面生成/分组应用 → 校验与精调 → 执行分析任务
- 结果查看器（Result）：选择任务 → 地图风险渲染 → 进度与错误明细 → 单断面矩阵详情

---

## 4/8 断面编辑器：数据加载与岸段选择

- 数据入口两类：上传岸段 GeoJSON；或从后端岸段库加载（`region_code` 分组 / `bank_id` 列表多选，支持删除）
- 地图选择两类：拾取岸段（支持全选/取消）；拾取段落起止点（更精确地约束范围）
- 规则配置三项：全局断面间距（m）、断面长度（m）、参数模板（后端拉取并可选）

---

## 5/8 断面生成、分组与批量规则应用

- 一键生成断面并创建任务：输入任务名 → 生成断面 → 创建 sections（并可继续“执行岸线分析”启动计算）
- 两种生成路径：生成“精细断面”（快速迭代）与生成“计算断面”（面向计算链路的增强流程）
- 自定义组（SelectionGroup）：对岸段子区间做差异化间距/长度/参数，并支持“一键应用”（必要时重绘断面）
- 岸段方向修正与同步：点击岸段批量反切断面并切换 `reversed`；支持把已选岸段（含修正标记）上传后端保持一致

---

## 6/8 断面精调与校验（编辑质量控制）

- 开启“断面精调”后可在地图上选中断面：岸段线模式支持沿岸线步进平移；自由模式支持拖拽/端点新建、旋转与缩放微调（并支持 Ctrl 多选）
- 断面常用动作齐全：反切方向、删除、属性配置；几何与参数可按需同步到后端
- 断面检查（validate）闭环：断面变化自动触发校验并可视化 pending/valid/invalid；也可手动强制重跑用于刷新/重试
- 视图与工具：断面“显示/隐藏设计”开关 + 一键清空当前编辑状态

---

## 7/8 导入导出与结果查看（任务闭环）

- 导入导出：上传断面 GeoJSON 直接创建任务/断面；导出断面样例 GeoJSON；参数弹窗支持导出 JSON 留痕复用
- 任务选择与地图渲染：加载任务列表、选择任务、断面图层显隐控制；按风险等级进行颜色映射展示
- 进度与错误追踪：轮询任务状态与结果统计（成功/失败/处理中），并支持展开查看单断面错误详情
- 深入查看：支持打开单断面“矩阵详情”，以表格/数组/JSON 的方式展示模型输出

---

## 8/8 架构设计（现状）与未来改进方向

- 架构分层：页面壳（聚合状态与导航）→ 业务动作模块 `src/pages/editor/*` → 组件层（地图/侧边栏/弹窗）→ 基础层（types/utils/services）
- 联调与部署：开发态用 Vite 代理 `/v0`；建议把 Mapbox token、后端地址等改为环境变量以支持多环境
- 稳定性与体验：抽象统一 API 客户端 + 错误提示；把“选择/编辑模式”收敛成明确状态机或 hooks，降低误操作
- 性能与工程化：大数据量 GeoJSON 的渲染/选择优化；补齐几何与结果解析的关键测试，并逐步清理 legacy 备份文件
