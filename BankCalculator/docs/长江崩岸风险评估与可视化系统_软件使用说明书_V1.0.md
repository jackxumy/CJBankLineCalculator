# 1. 软件概述

本系统用于面向岸线（岸段）与断面（横断面）开展崩岸风险评估的任务创建、断面几何编辑、参数配置、模型计算启动与结果可视化。

系统整体分为两部分：

- 前端可视化与交互：地图展示（Mapbox GL JS）+ 断面编辑 + 任务与结果查看。
- 后端计算与数据服务：以 `/v0/bank/*` 形式提供岸段、断面、参数模板、任务与结果等接口（本文仅描述前端使用方式）。

【截图占位：系统整体界面（首页/编辑器/结果页）】

## 1.1 运行环境

### 1.1.1 硬件建议

- 客户端
  - CPU：双核及以上
  - 内存：8GB 及以上（建议）
  - 显卡：集显可用；断面较多时建议独显
- 服务器（后端，若由本单位部署）
  - 由后端计算规模决定；建议与后端团队确认

### 1.1.2 软件建议

- 客户端
  - Windows 10/11（或 macOS/Linux）
  - 现代浏览器：Chrome / Edge（建议使用最新版）
- 网络
  - 需要能访问后端接口（同源或由代理转发）

## 1.2 系统开发环境

（基于当前前端工程配置整理）

- 前端
  - Node.js：建议 18+（Vite 7 / React 19 组合一般要求较新 Node 版本）
  - 框架：React + TypeScript
  - 构建工具：Vite
  - GIS：Mapbox GL JS + Turf.js
- 后端（接口侧）
  - 不在本前端仓库内；需提供 `/v0/bank/*` 接口

## 1.3 系统部署环境

- 前端部署方式
  - 通过 `npm run build` 产出静态资源（`dist/`），可部署到 Nginx/Apache/静态资源服务器
- 后端连通性
  - 生产环境需确保前端访问 `/v0/*` 能正确转发到后端服务
  - 开发环境下，前端已在 Vite 中配置了 `/v0` 代理（详见“附录-常见问题”）

## 2.3 主要功能（断面编辑器）

断面编辑器界面由两部分组成：

- 地图区域：展示岸段、断面、选择点、校验状态与编辑交互
- 左侧控制面板：数据加载、断面规则、分段配置、断面精调、分析启动

【截图占位：断面编辑器整体界面】

### 2.3.1 数据加载（岸段/断面）

在左侧面板“数据加载与配置”中，提供两种导入方式：

1) 上传岸段（GeoJSON）
- 按钮：上传岸段
- 支持格式：`.geojson` / `application/json`
- 作用：将岸段线要素加载到地图，用于后续选段、生成断面。

2) 上传断面（GeoJSON）
- 按钮：上传断面
- 作用：将已有断面线要素导入，并在后端创建“导入断面任务”与断面资源。
- 说明：导入时会提示输入任务名称。

【截图占位：上传岸段/上传断面按钮】

### 2.3.2 岸段获取与删除（后端资源）

系统支持从后端加载已存储的岸段资源，并可删除。

- 下拉框：获取岸段（选择岸段/ID）
  - 优先按 `bank_id` 列出可选岸段
  - 选择后自动加载该岸段到编辑器
- 删除按钮：删除
  - 当选择的是单个 `bank_id` 时：删除该岸段
  - 当选择的是 `region_code`（遗留/兼容逻辑）时：按组批量删除

【截图占位：获取岸段下拉框与删除按钮】

### 2.3.3 断面规则配置（间距/长度）

在“岸段与断面规则”区域配置生成断面的全局规则：

- 断面间距（m）：默认 100
- 断面长度（m）：默认 1000

该规则用于“生成全部断面”，也作为后续“段落起止控制”的初始参数。

【截图占位：断面间距/长度输入框】

### 2.3.4 岸段选择与全选

生成断面前，需要明确用于分析的岸段范围。

- 拾取岸段
  - 点击后进入“选择中”状态
  - 在地图上逐条点击岸段可进行多选/取消
- 全选/取消
  - 一键选择当前已加载的全部岸段线要素

提示：面板会显示“要素数 / 已选岸段数 / 断面数”。

【截图占位：拾取岸段/全选按钮与选段高亮】

### 2.3.5 生成断面并创建任务

按钮：生成全部断面

操作流程：

1) 上传或获取岸段
2) 通过“拾取岸段/全选”选中要分析的岸段
3) 点击“生成全部断面”
4) 输入任务名称（弹窗提示）
5) 系统将：
   - 创建任务（后端 `/v0/bank/tasks`）
   - 沿选中岸段按间距生成断面几何
   - 创建断面资源（后端 `/v0/bank/sections`）

生成完成后，地图会显示断面线，且每条断面会带 `sectionId` 等标识用于后续校验与计算。

【截图占位：生成全部断面按钮与生成后的断面展示】

### 2.3.6 断面校验（断面检查）

生成/导入断面后，系统会自动触发断面校验；同时提供手动“断面检查”按钮用于强制重新校验。

- 按钮：断面检查
- 校验接口：`POST /v0/bank/sections/{sectionId}/validate`
- 断面颜色含义（编辑器地图）：
  - 绿色：通过（valid）
  - 红色：未通过（invalid）
  - 黄色：校验中/未知/请求失败（pending）

提示：点击断面可弹出校验状态信息。

【截图占位：断面校验状态颜色与弹窗】

### 2.3.7 岸段方向修正（修正选择/发送）

当发现某些岸段断面生成方向需要整体反向时，可使用“岸段方向修正”。

- 前提：已生成断面，且已选中需要修正的岸段
- 按钮：修正选择
  - 开启后，地图点击某条“已选岸段”将触发：
    - 该岸段下所有断面几何反转
    - 并将岸段标记 `properties.reversed=true`
    - 若断面存在 `sectionId`，会尝试同步后端反转
- 按钮：发送
  - 将“已选岸段”的 GeoJSON（包含 reversed 标记，且 MultiLineString 会拆分成多个 LineString）发送到后端岸段资源

【截图占位：修正选择/发送按钮】

### 2.3.8 属性配置（全局/组/单断面）

系统支持为断面计算配置属性参数（SectionParams），并可同步到后端。

1) 参数模板（可选）
- 下拉框：参数模板
- 选择后会从后端拉取模板详情，并作为当前“全局属性配置”的初始值。

2) 全局属性配置
- 按钮：属性配置（位于“岸段与断面规则”下）
- 用途：配置整体任务默认参数（如时间点、潮位、风险阈值、权重等）

3) 组属性配置 / 单断面属性配置
- 在“段落起止控制”的段落条目中点“属性”可配置该段落的属性（需再点击“应用”使其落到断面上）
- 在“断面精细调整”选中断面后点“属性”可配置单条断面属性，并可通过 `PUT /v0/bank/sections/{id}` 保存

【截图占位：属性配置弹窗（基础信息/DEM/水深/阈值/权重）】

### 2.3.9 段落起止控制（自定义区段）

用于对同一岸段的不同里程区间设置不同的断面间距/长度/属性。

- 按钮：拾取段落位置
  - 开启后，在地图上点击岸段：
    - 第一次点击：设置起点
    - 第二次点击（同一条线）：设置终点
  - 若第二次点击在另一条岸段线上，将自动开始新段落

生成的段落会在面板中以“段落 1/2/…”列出，并在地图上显示起止点与区段线。

【截图占位：拾取段落位置 + 地图起止点/区段显示】

### 2.3.10 应用区段配置与反切

对某段落展开后，可进行：

- 修改间距/长度
- 属性：配置该段落的属性
- 反切：反转该段落范围内的断面方向（并尝试同步后端）
- 应用：将该段落的配置应用到断面
  - 若仅修改长度：只调整已有断面长度
  - 若修改间距：会删除该区段原有断面并按新间距重绘

【截图占位：段落配置（间距/长度/属性/反切/应用）】

### 2.3.11 断面精细调整（选择/添加/平移/旋转/缩放/删除）

用于在生成断面后进行更精细的几何编辑。

- 按钮：开启断面精调 / 完成编辑

开启后可选择控制模式：

1) 岸段线模式
- 添加：点击岸段某位置生成断面（按当前全局长度）
- 平移：对选中断面进行 -5m/-1m/+1m/+5m 沿岸线方向移动

2) 自由模式
- 选择：可点击断面并拖拽整体移动
- 旋转：±5°
- 缩放：±10m（拉长/缩短）
- 添加：通过“点选起点→点选终点”创建一条断面

对选中断面还可执行：
- 删除
- 反切（反转方向）
- 属性（编辑参数并可保存到后端）

【截图占位：断面精调区域（岸段线/自由、选择/添加、平移/旋转/缩放、断面属性）】

### 2.3.12 视图与工具（隐藏设计/清空）

- 隐藏设计/显示设计：控制断面图层显隐
- 清空：清空当前分段信息与断面数据（不影响后端已创建的任务/断面资源）

【截图占位：隐藏设计/清空】

# 3. 附录

## 3.1 数据格式要求（GeoJSON）

### 3.1.1 岸段（上传岸段）

- 支持：`FeatureCollection` / 单个 `Feature`
- 几何类型：`LineString` / `MultiLineString`
- 建议属性（properties）：
  - `bank_id`：岸段唯一标识（建议提供；否则会按索引生成）
  - `bank_name`：岸段名称（可选）
  - `region_code`：区域编码（可选，默认会按 Mzs 处理）
  - `reversed`：是否需要反向生成断面（true/"true"）

注意：
- MultiLineString 在“发送”到后端时会被拆分为多条 LineString，并追加 `_partN` 后缀。

【截图占位：岸段 GeoJSON 示例（可放一段示例文本或截图）】

### 3.1.2 断面（上传断面）

- 输入：GeoJSON 断面线集合（LineString）
- 推荐属性（可选）：
  - `section_id`：断面ID
  - `section_name`：断面名称
  - `bank_id`：所属岸段ID
  - `distance`：在母线上的里程（单位 m；无则按 0 处理）

【截图占位：断面 GeoJSON 示例】

## 3.2 常见问题与排查

1) 无法加载任务/断面/参数模板
- 现象：任务列表为空、加载失败、提示无法加载
- 排查：确认后端服务可访问，且前端 `/v0/*` 请求已正确转发

2) 开发环境接口代理不通
- 本工程开发代理配置在 `vite.config.ts`：`/v0` → `http://192.168.1.102:8090`
- 若后端地址不同，请在部署/开发时调整代理或使用反向代理保持同源

3) 断面一直黄色（pending）或校验失败
- 可能原因：校验接口不可用/网络不通/后端未返回 JSON
- 建议：点击“断面检查”强制重试；打开浏览器控制台查看网络错误

4) 执行岸线分析按钮不可用或提示校验未通过
- 原因：分析前强制要求所有断面校验通过（绿色）
- 处理：先对红色断面进行“反切/平移/旋转/缩放/删除/重建”，直至校验通过
